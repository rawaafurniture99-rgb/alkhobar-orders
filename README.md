# Al Khobar Orders — تجهيز التطبيق كـ PWA

اللي اتعمل في الملفات دي:
- `index.html` — نفس البرنامج بتاعك + إضافات PWA (manifest, أيقونات, service worker)
- `manifest.json` — بيانات التطبيق (الاسم، الألوان، الأيقونات) اللي بتخلي المتصفح يعرض "إضافة إلى الشاشة الرئيسية"
- `sw.js` — service worker بسيط يخزن شكل التطبيق (الواجهة) عشان يفتح بسرعة حتى مع نت ضعيف، من غير ما يأثر على بيانات Firebase اللي لازم تفضل لايف
- `icons/` — أيقونات التطبيق بمقاسات مختلفة، مأخوذة من لوجو رواعة فرنتشر

## خطوة مهمة جدًا قبل النشر
البرنامج فيه صلاحية دخول تعتمد على إيميلات مسموح لها (Access/Team management) — ده كويس ومناسب. بس لاحظي إن مفتاح Firebase API الظاهر في الكود (`apiKey`) شيء طبيعي إنه يكون ظاهر في تطبيقات الويب، لكن **لازم تتأكدي إن Firestore Security Rules عندك مضبوطة صح** بحيث محدش غير المسموح لهم يقدر يقرأ/يعدل البيانات، حتى لو حصل على الرابط. لو عايزة أراجعلك الـ Rules قوليلي.

## إزاي تنشريها (لازم رابط HTTPS عشان الـ PWA تشتغل وتتثبت من الموبايل)

اختاري واحدة من دول (كلها مجانية):

### الخيار الأسهل: Firebase Hosting (لأنك أصلًا مستخدمة Firebase)
```bash
npm install -g firebase-tools
firebase login
firebase init hosting   # اختاري نفس مشروع al-khobar-orders، ومجلد public = هذا المجلد
firebase deploy
```

### أو Netlify (سحب وإفلات)
1. روحي على https://app.netlify.com/drop
2. اسحبي المجلد ده كله (بالملفات والمجلدات جوه) وارميه في الصفحة
3. هتاخدي رابط جاهز فورًا

## إزاي تحمّليها على الموبايل بعد النشر
- **أندرويد (Chrome)**: افتحي الرابط → هيظهر إشعار "إضافة إلى الشاشة الرئيسية" أو من قائمة الثلاث نقط → "تثبيت التطبيق"
- **آيفون (Safari)**: افتحي الرابط → زر المشاركة (Share) → "إضافة إلى الشاشة الرئيسية"

بعد كده هتلاقي أيقونة رواعة فرنتشر على شاشة الموبايل وتفتح زي أبلكيشن عادي بدون شريط المتصفح.
